# ConcurrentHashMap 存储并发长连接是否存在性能瓶颈？



> ConcurrentHashMap 存储并发长连接不仅保障了并发安全，而且不会导致性能瓶颈。这篇博客带你分析原因。



几乎不存在性能瓶颈。

在长连接场景下，ConcurrentHashMap 通常只负责「维护连接关系」。可以从读操作、写操作两个方向展开分析。



## 读操作

**读操作的例子**：对于「长连接场景」下的读操作，大部分是在「**根据连接 ID 查询连接**」。

ConcurrentHashMap 的 `get()` 没有锁，没有同步开销，不会阻塞。所以不存在性能瓶颈。



## 写操作

**写操作的例子**：长连接的「建立连接、断开连接」都属于写操作。

对于 JDK8，每把锁只锁住一个桶，不会影响其他桶，所以它的性能很高（相比于 HashTable）。
