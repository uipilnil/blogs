# 为什么用到泛型的静态方法要提前声明 < T >



> 如果方法的返回值类型用到了泛型，对于静态方法，需要在前面声明 < T >；对于非静态方法，不需要提前声明。本篇博客将会分析这样设计的原因。



## 描述现象

如果「方法的返回值类型用到了泛型」：

- 对于静态方法，需要在前面声明 `<T>`；
- 对于非静态方法，不需要提前声明。

```java
public static <T> Result<T> success(T data) {
    Result<T> result = new Result<>();

    result.setCode(200);
    result.setMsg("操作成功");
    result.setData(data);

    return result;
}
```



## 分析原因

`T` 取什么类型，是在 **创建实例对象** 的时候确定的。

- 当 `new Result<String>` 时，`T` 取 `String`；
- 当 `new Result<Integer>` 时，`T` 取 `Integer`。

在调用 **非静态方法** 时，方法会从 **对象的信息** 中读取到 `T` 的类型，用这个类型把 `T` 替换掉，方法可以正常执行。

```java
public Result<T> success(T data) {}
```

然而，**静态方法** 是属于类的，调用静态方法不需要创建对象。

所以，在调用静态方法时，并不知道 `T` 是什么。

为了解决这个问题，要回到定义静态方法的时候。在方法签名中，声明泛型 `<T>`，提前和方法“打好招呼”。

这个 `<T>` 是独立的，它和类的泛型 `<T>` 没有任何关系，为了方便区分，可以叫它泛型 `<U>`。

```java
public class Result<T> {
    private int code;
    private String msg;
    private T data;

    public static <U> Result<U> success(U data) {
        Result<U> result = new Result<>();

        result.setCode(200);
        result.setMsg("操作成功");
        result.setData(data);

        return result;
    }
}
```
